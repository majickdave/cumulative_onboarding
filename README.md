# Cumulative Onboarding

A repository of onboarding materials, exercises, templates, and small example projects used to onboard new team members at Mindtree. It collects guides and hands‑on content that progressively introduces systems, tools, processes, and code patterns used across the organization.

## What’s in this repo
- data/ — extracts downloaded daily
- README.md — this overview

## Goals
- Provide consistent reporting for topline metrics like appointments, revenue
- Centralize templates and examples to reduce ramp time
- Offer documentation that scales across roles (dev, infra, QA, PM)

## Getting started
1. Clone the repo:
    git clone <repo-url>
2. Inspect the top‑level folders and open docs/ or exercises/ relevant to your role.
3. Follow per‑folder README files for step‑by‑step instructions and prerequisites.

## Common workflows
- Read the role or track guide under docs/ to find a recommended learning path.
- Complete exercises/ tasks and optionally submit a draft PR using templates/PR_TEMPLATE.md.
- Use scripts/ for environment setup or checks (see each script’s header or docs/ for usage).

## Contributing
- Follow the templates in templates/ when creating issues or PRs.
- Keep changes small and focused; include a reference to the onboarding step or exercise you’re improving.
- Run any available lint or test scripts before submitting.

## Maintenance notes
- Keep examples small and self‑contained.
- Document assumptions and environment requirements inside folder READMEs.
- Version exercises when external dependencies or APIs change.

## Contact
For questions or to propose new material, open an issue or reach out to the onboarding maintainer team listed in docs/MAINTAINERS.md.
